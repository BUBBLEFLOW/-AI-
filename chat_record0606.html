1. 项目说明与输入输出

用户
解释这个项目，说明输入输出。
Codex

项目是 Ambisonizer，当前代码最初是 stereo -> FOA WXYZ。训练输入为双声道 stereo.wav，目标输出为一阶 Ambisonic B-format 的四个通道：

输入: stereo.wav, shape [B, 2, T]
输出: W.wav / X.wav / Y.wav / Z.wav, shape [B, 4, T]
采样率: 24000 Hz
默认长度: 120000 samples, 即 5 秒

模型结构在 model/seanet.py 中：双声道输入先平均成 mono 做主编码，同时双声道 conditioner 提供条件信息，最后 decoder 输出四通道 W/X/Y/Z。

2. 原 stereo 数据集设计是否合理

用户
这个数据集生成合理吗？整体思路有什么问题吗？
Codex

结论：工程流程自洽，但作为真实 stereo -> FOA 数据集不够合理。原生成链是：

FOA RIR: W/X/Y/Z
  -> 固定 ±30° 虚拟扬声器解码成 stereo RIR
  -> mono clean speech 卷积 stereo RIR 得到输入 stereo.wav
  -> mono clean speech 卷积原 FOA RIR 得到目标 W/X/Y/Z

主要问题：

  • 2 通道 stereo 无法唯一恢复 4 通道 FOA，是欠定问题。
  • 默认 elevation=0 时，Z 通道没有进入输入，模型只能猜。
  • W/X 也不可唯一分离，输入里只有混合项。
  • 训练/测试同一合成管线，容易只测试插值能力。
  • train/test split 没有按 room / clean speaker / RIR group 严格隔离，可能泄漏。

3. Binaural -> FOA 的合理性

用户
如果是 binaural 转 FOA 合理吗？能还原出四声道吗？
Codex

binaural -> FOA 比普通 stereo -> FOA 更合理，因为 binaural 包含 HRTF 线索：

  • ITD：左右时间差，提供水平方向信息。
  • ILD：左右强度差，提供方向和频率相关线索。
  • 耳廓频谱变化：提供 elevation，也就是 Z 相关线索。

但它仍然不是严格可逆：输入只有 L/R 两通道，目标是 W/X/Y/Z 四通道。模型生成的是基于训练数据先验的合理 FOA 估计，不是数学上精确还原。

5. Flow Matching 讨论

用户
这个网络现在是什么？如果想用 Flow Matching 可以把这个网络加到哪里？
Codex

当前网络本质是 条件 SEANet U-Net + Transformer + VAE 风格条件分支，一次前向直接回归 WXYZ。

Flow Matching 可改为 conditional velocity network：

训练:
y = target WXYZ
z = random noise
t ~ Uniform(0, 1)
x_t = (1 - t) * z + t * y
target_velocity = y - z
pred_velocity = model(x_t, t, condition)

结构上要加 4 通道 x_t encoder 和 time embedding，decoder 输出 velocity WXYZ。更稳的做法是先做 Flow Matching refiner。

8. BRIR / FOA RIR 听起来没声音的问题

用户
现在的这个根本就没有声音 input_binaural。生成的 binaural 就没声音，是因为 HRTF 没有归一化吗？现在 BRIR 还是没有声音。
Codex

诊断结论：

  • BRIR 是脉冲响应，不是语音，直接听本来就很短，不像连续声音。
  • 用户提供的 binaural(1).wav 经检查不是全零，但电平很低：
采样率: 24000 Hz
通道数: 2
长度: 0.305 秒
左声道 peak: 0.005887
右声道 peak: 0.011076
整体 peak: 0.011076
整体 RMS: 0.000235
peak 约 -38 dBFS
RMS 约 -65 dBFS

因此问题是 BRIR 输出幅度太低或使用了旧文件，没有被归一化到 0.95。

已做改动：

  • 03_generate_binaural_rirs.py 默认 NORMALIZE_PEAK = True。
  • 新增 peak_after_normalize、rms_after_normalize metadata。
  • 新增 binaural_audition.wav，用于试听 BRIR 的 impulse 响应，默认写成 PCM_16。
  • 04_generate_ears_stereo_audio_train_test.py 默认按 input binaural 峰值归一化，避免 input 被 target WXYZ 压得太小。

建议重跑：

python tau_srir_pipeline/03_generate_binaural_rirs.py --overwrite --normalize_peak
python tau_srir_pipeline/04_generate_ears_stereo_audio_train_test.py --overwrite
用户
现在的 TAU-SRIR 有没有归一化？现在的 HRTF 有没有归一化？为啥现在声音听起来特别小，基本听不到 BRIR？
Codex

TAU-SRIR FOA RIR：第 02 步默认归一化，NORMALIZE_PEAK=True，四个通道共享一个 gain，不单独归一化每个通道。

HRTF/HRIR：没有单独归一化。代码直接读取 SOFA Data.IR，用它生成 FOA-to-binaural filters，然后对生成出来的 BRIR 结果做归一化。这样避免破坏 HRTF 的左右耳相对幅度、频谱和方向线索。

声音小的原因：用户检查的 BRIR 文件 peak 只有约 0.011，说明很可能是旧文件、没覆盖生成、或听的不是新生成文件。

用户
