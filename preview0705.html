核心结论速览

  • 严格输入只能是 stereo 时，真正可作为学习式 baseline 的开源方法较少，最接近的是 Ambisonizer。
  • 传统 DSP baseline 推荐使用 IEM StereoEncoder 或自己实现 IEM-like fixed-width FOA encoder。
  • IEM StereoEncoder 的本质是：把 stereo 左右声道看作两个固定方位的虚拟声源，再用一阶 Ambisonics 球谐编码得到 FOA。
  • 实验中建议固定 Azimuth=0°、Elevation=0°、Gain=0 dB，只改变 Width=60°/90°/120°。
  • IEM 默认输出为 AmbiX / ACN / SN3D，一阶通道顺序是 W, Y, Z, X，如目标是 W, X, Y, Z，需要重排。

3. IEM StereoEncoder 作为 DSP baseline 的原理

用户：DSP baseline：IEM StereoEncoder。这个很适合做传统方法 baseline。输入就是 stereo，输出设成一阶 Ambisonics。可以设几组参数：IEM-StereoEncoder-60deg、90deg、120deg。这个原理是啥？

AI：核心原理是：把 stereo 的 L/R 两个声道假设成空间中的两个虚拟点声源，然后分别编码成 FOA，最后相加。

输入：L(t), R(t)

Width = 60°：
L → +30°
R → -30°

Width = 90°：
L → +45°
R → -45°

Width = 120°：
L → +60°
R → -60°

FOA 四个通道可理解为：

W = 全向分量，整体能量
X = 前后方向分量
Y = 左右方向分量
Z = 上下方向分量

公式上：

FOA(t) = Encode(L(t), left_direction) + Encode(R(t), right_direction)

B_i(t) = L(t) * SH_i(left_direction) + R(t) * SH_i(right_direction)

其中 SH_i 是对应方向上的一阶球谐函数权重。

这个方法不会理解音乐内容，也不会恢复真实复杂声场。它只是一个固定几何假设，非常适合作为传统 DSP baseline。

4. IEM StereoEncoder 在 REAPER 中如何使用

用户：IEM StereoEncoder 到底是咋用的？

AI：目标流程是：输入 stereo wav，输出 4 声道 FOA wav。

  1. 安装 IEM Plug-in Suite。
  2. 打开 REAPER，把 stereo wav 拖入。
  3. 点击该 track 的 Route / I/O 按钮。
  4. 把 Track channels 从 2 设置为 4。
  5. 在该轨道添加 FX，搜索并插入 IEM StereoEncoder。
  6. 设置参数：Order=1st、Normalization=SN3D、Azimuth=0°、Elevation=0°、Roll=0°、Width=60°/90°/120°。
  7. 渲染时选择 WAV，Channels 设置为 4。

重要注意：IEM 输出为 AmbiX / ACN / SN3D。一阶 ACN 顺序是：

IEM 输出顺序 = W, Y, Z, X

如果模型 target 是 W, X, Y, Z，需要重排：

# iem: [W, Y, Z, X]
# target: [W, X, Y, Z]
foa_wxyz = iem[:, [0, 3, 1, 2]]

5. 能不能直接自己编程实现

用户：这个能不能直接自己编程用？

AI：可以。甚至更推荐直接自己写一个 IEM-like StereoEncoder baseline，而不是在实验管线里调用插件。

核心思想：

输入：L(t), R(t)
width = 60° / 90° / 120°
φ = width / 2

L: azimuth = +φ, elevation = 0°
R: azimuth = -φ, elevation = 0°

一个简单实现：

import numpy as np
import soundfile as sf


def stereo_to_foa_fixed_width(input_wav, output_wav, width_deg=60, output_order="WXYZ", normalize=True):
    audio, sr = sf.read(input_wav, always_2d=True)
    if audio.shape[1] != 2:
        raise ValueError(f"Expected stereo input with 2 channels, got {audio.shape[1]} channels.")

    L = audio[:, 0]
    R = audio[:, 1]
    theta = np.deg2rad(width_deg / 2.0)

    W = 0.5 * (L + R)
    X = 0.5 * np.cos(theta) * (L + R)
    Y = 0.5 * np.sin(theta) * (L - R)
    Z = np.zeros_like(W)

    if output_order.upper() == "WXYZ":
        foa = np.stack([W, X, Y, Z], axis=1)
    elif output_order.upper() == "WYZX":
        foa = np.stack([W, Y, Z, X], axis=1)
    else:
        raise ValueError("output_order must be 'WXYZ' or 'WYZX'.")

    if normalize:
        peak = np.max(np.abs(foa))
        if peak > 1.0:
            foa = foa / peak * 0.999

    sf.write(output_wav, foa, sr)

这种方法更适合批量跑数据集，也更容易在论文中复现。

7. 这个方法如何将 stereo 编码到 FOA

用户：这个方法是如何将 stereo 编码到 FOA 的？

AI：本质是：把 stereo 的 L/R 两个声道看成两个固定方位的虚拟声源，然后用 FOA 球谐编码公式投影成 W/X/Y/Z 四个通道。

φ = width / 2

FOA(t) = Encode(L, +φ, 0°) + Encode(R, -φ, 0°)

展开：

W = L + R
X = L * cos(+φ) + R * cos(-φ)
Y = L * sin(+φ) + R * sin(-φ)
Z = 0

由于：

cos(+φ) = cos(-φ)
sin(-φ) = -sin(+φ)

所以可化简为：

W = L + R
X = cos(φ) * (L + R)
Y = sin(φ) * (L - R)
Z = 0

实际工程中常加 0.5 归一化：

W = 0.5 * (L + R)
X = 0.5 * cos(φ) * (L + R)
Y = 0.5 * sin(φ) * (L - R)
Z = 0

所以 width 越大，左右分量 Y 越强，前向分量 X 越弱。

8. IEM StereoEncoder 的参数含义

用户：这个每个 Encoder setting 调整 Azimuth、Elevation、Gain 是啥意思？

AI：这些不是生成模型参数，而是虚拟声源的空间摆放参数。

参数含义实验建议
Azimuth方位角，控制水平方向。0° 是正前方，+90° 是左边，-90° 是右边，180° 是后方。设为 0°
Elevation高度角，控制上下方向。0° 是水平面，+90° 是正上方，-90° 是正下方。设为 0°
Gain音量增益，控制输出响度。设为 0 dB
Width控制 stereo L/R 两个输入声道分开的角度。设 60° / 90° / 120°

推荐实验设置：

Azimuth = 0°
Elevation = 0°
Gain = 0 dB
Roll = 0°
Order = 1st
Normalization = SN3D
Width = 60° / 90° / 120°

10. 可直接写入论文的方法描述

We implement an IEM-StereoEncoder-like DSP baseline by treating the left and right stereo channels as two virtual point sources placed symmetrically around the frontal direction. For a stereo width θ, the two channels are positioned at azimuths +θ/2 and −θ/2 with zero elevation, and are encoded into first-order Ambisonics using fixed first-order spherical harmonic weights. We evaluate three widths: 60°, 90°, and 120°.

中文：

我们实现了一个类似 IEM StereoEncoder 的传统 DSP baseline。该方法将 stereo 的左右声道视为位于正前方两侧的两个虚拟点声源。对于给定 stereo width θ，左右声道分别放置在 +θ/2 和 −θ/2 的方位角上，高度角为 0，并通过固定的一阶球谐权重编码为 FOA。实验中测试 60°、90° 和 120° 三种宽度。
