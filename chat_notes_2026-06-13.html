2. stereo/FOA 生成逻辑结论

当前 tools/generate_stereo_foa_dataset.py 的每条样本只选择一个固定位置：

location = rng.choice(locations)
foa_rir = foa[:, :, location.doa_index]

因此每个 sample_xxxxxx.wav 是：

  • 固定房间 room
  • 固定麦克风/接收点 ref_index
  • 固定扬声器/声源方向 doa_index

这不是连续运动轨迹。所以最终文本应描述为“声音稳定地从某方向传来”，不应写“从左到右移动”“靠近/远离”“绕着听者运动”。

3. Caption 脚本

新增并多次修改了：

tools/generate_stereo_foa_captions.py

它现在支持：

  • 从 metadata.csv 读取映射，或通过 --rebuild_metadata 从 EARS/TAU-SRIR 重建映射。
  • 根据 FOA RIR WXYZ 估计方向词，例如前方、后方、左侧、右侧、上方、下方。
  • 生成每个音频对应的双语 txt。
  • 调用 Qwen / OpenAI-compatible API 生成自然语言描述。
  • 对 Qwen 输出做后处理：强制补回房间描述，移除房间编号和通风噪声。

模板版输出路径

data\stereo_foa\captions_txt
data\stereo_foa\captions_manifest.csv

Qwen 版输出路径

data\stereo_foa\captions_txt_qwen
data\stereo_foa\captions_manifest_qwen.csv

5. Qwen 生成过程

用户在 PowerShell 中设置了 DashScope API Key。本文档不保存密钥，命令中只保留占位符：

$env:DASHSCOPE_API_KEY="[REDACTED]"

Qwen 生成命令

cd C:\Users\25637\Desktop\DiffStereo-main___stereo_WXYZ___FM0611

python tools/generate_stereo_foa_captions.py `
  --dataset_root data/stereo_foa `
  --rebuild_metadata `
  --ears_root D:/B2FOA/EARS `
  --srir_root D:/B2FOA/TAU-SRIR/TAU-SRIR_DB `
  --output_dir data/stereo_foa/captions_txt_qwen `
  --manifest data/stereo_foa/captions_manifest_qwen.csv `
  --variants_per_file 1 `
  --estimate_direction_from_rir `
  --llm_endpoint https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions `
  --llm_model qwen-plus `
  --llm_api_key_env DASHSCOPE_API_KEY `
  --strict_llm

本次实际生成模型为 qwen-plus。不是本地 qwen2.5:7b。

Qwen 输出后处理

Qwen 初始输出有时漏掉房间描述，只保留方向。后来用 repair 模式强制补回房间描述，并清理房间编号和通风噪声：

python tools/generate_stereo_foa_captions.py `
  --dataset_root data/stereo_foa `
  --repair_manifest data/stereo_foa/captions_manifest_qwen.csv `
  --output_dir data/stereo_foa/captions_txt_qwen `
  --manifest data/stereo_foa/captions_manifest_qwen.csv

7. 研究思路讨论

当前 baseline 是：

stereo -> FOA WXYZ

这可以学习 stereo 到 FOA 的映射，但 stereo 本身对前后、上下、房间语义等信息表达不完整。

更有研究价值的扩展是：

text + stereo -> FOA WXYZ

文本可以提供：

  • 房间类型和材质语义
  • 声源方向
  • 前后/上下等 stereo 中不稳定的信息
  • 面向可控空间音频生成的条件

建议实验路线：

  1. 先做 stereo -> FOA baseline。
  2. 再做 stereo + text -> FOA。
  3. 训练时使用 text dropout，例如 10%-20%，避免模型完全依赖文本。
  4. 指标上比较 FOA 通道相关性、stereo consistency、方向一致性和主观听感。
