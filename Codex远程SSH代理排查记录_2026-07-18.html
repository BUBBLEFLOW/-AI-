1. 问题现象

Codex 启动后反复出现：

Reconnecting...
Request timed out

或者：

Falling back from WebSockets to HTTPS transport
Connection refused (os error 111)
stream disconnected before completion

这些通常表示远程服务器无法直接访问 OpenAI，或者 Codex 没有正确使用代理。

2. 确认 Windows 本地代理

已确认本地监听端口：

127.0.0.1:7890  HTTP 代理
7891            SOCKS 代理

Windows PowerShell 测试命令：

curl.exe -v -x http://127.0.0.1:7890 https://api.openai.com

出现以下内容说明本地代理正常：

HTTP/1.1 200 Connection established
Welcome to the OpenAI API!

结论：Windows 本地 Clash 代理工作正常。

3. SSH 配置

配置文件位置：

C:\Users\qpyl\.ssh\config

配置内容：

Host autodl-westb
  HostName connect.westb.seetacloud.com
  Port 20661
  User root
  RemoteForward 17890 127.0.0.1:7890
  ExitOnForwardFailure yes
  ServerAliveInterval 30
  ServerAliveCountMax 3

Host campus-server
  HostName 202.116.7.104
  Port 2230
  User liuwf
  RemoteForward 17890 127.0.0.1:7890
  ExitOnForwardFailure yes
  ServerAliveInterval 30
  ServerAliveCountMax 3

验证 SSH 是否读取到配置：

ssh -F "$env:USERPROFILE\.ssh\config" -G autodl-westb |
  findstr /i "hostname port user remoteforward"

正确结果应包含：

user root
hostname connect.westb.seetacloud.com
port 20661
remoteforward 17890 [127.0.0.1]:7890

4. 建立 SSH 反向端口转发

配置含义：

AutoDL 容器 127.0.0.1:17890
↓ SSH 隧道
Windows 本机 Clash 127.0.0.1:7890
↓
OpenAI API

Windows PowerShell 中建立连接：

ssh autodl-westb

排查时可以使用详细日志：

ssh -vv autodl-westb

SSH PowerShell 窗口不能关闭，关闭后隧道也会断开。

远程端测试：

curl -v -x http://127.0.0.1:17890 --connect-timeout 10 https://api.openai.com

成功标志：

Connected to 127.0.0.1 port 17890
channel 1: connected to 127.0.0.1 port 7890
HTTP/1.1 200 Connection established
SSL certificate verify ok
Welcome to the OpenAI API!

结论：SSH 隧道已经成功打通。

5. 远程服务器设置代理变量

每次新开 SSH 会话后，先执行：

export HTTP_PROXY=http://127.0.0.1:17890
export HTTPS_PROXY=http://127.0.0.1:17890
export http_proxy=http://127.0.0.1:17890
export https_proxy=http://127.0.0.1:17890

检查：

env | grep -i proxy

不使用 -x 再测试一次：

curl -v --connect-timeout 15 https://api.openai.com

永久保存到 ~/.bashrc：

cat >> ~/.bashrc <<'EOF'
export HTTP_PROXY=http://127.0.0.1:17890
export HTTPS_PROXY=http://127.0.0.1:17890
export http_proxy=http://127.0.0.1:17890
export https_proxy=http://127.0.0.1:17890
EOF

source ~/.bashrc

7. “Quota exceeded” 的含义

Quota exceeded. Check your plan and billing details.

这表示网络已经通了，但当前登录账号的 Codex 配额不可用，常见原因：

  • Codex 登录的不是购买 Plus 的那个 ChatGPT 账号。
  • Plus 套餐内 Codex 使用额度已经达到当前周期上限。
  • 当前使用的是 API Key 登录，API 余额不足或未启用计费。

Context compacted 只是 Codex 压缩长会话上下文，不是错误。

8. 重新登录 Codex

退出当前登录：

codex logout

使用 ChatGPT 账号设备码登录：

codex login --device-auth

然后：

  1. 在 Windows 浏览器打开终端显示的网址。
  2. 登录购买 Plus 的同一个 ChatGPT 账号。
  3. 输入终端生成的一次性验证码。
  4. 授权完成后返回终端。

检查登录状态：

codex login status

安全提示：一次性验证码不要发给别人。原对话中出现的验证码已在本文件中脱敏，不予保留。

9. 其他提示

ss: command not found

表示系统未安装 ss，通常由 iproute2 提供：

apt-get update
apt-get install -y iproute2

bubblewrap 警告

Codex could not find bubblewrap on PATH

这只是沙箱依赖警告。Codex 会暂时使用内置 bubblewrap，不是本次联网失败的原因。

nvidia-smi: No devices were found

表示当前容器没有识别到 GPU，与 Codex 代理问题无关。

最终检查清单

  1. Windows 本地 127.0.0.1:7890 可以访问 OpenAI。
  2. SSH 使用 autodl-westb 配置连接。
  3. 远程 127.0.0.1:17890 能通过隧道访问本地 7890。
  4. 远程终端的代理环境变量存在。
  5. Codex 登录的是购买 Plus 的同一个 ChatGPT 账号。
