用户 Failed to fetch

Generate
10:47:43 Submitting task...
10:47:45 Failed to fetch

助手

判断 Failed to fetch 是浏览器没有连到 http://localhost:8001 的 API。

排查后发现 API 启动失败原因包括：

  • Triton 想写入 C:\Users\25637\.triton，权限被拒。
  • 项目内 .cache\acestep\tmp 目录无法创建。

新增了 start_studio_api.bat，让 API 使用项目内 runtime_cache/，并在 acestep/api_server.py 中支持 ACESTEP_CACHE_ROOT。

用户 Model not initialized

Generate
11:02:37 Submitting task...
11:02:37 Task ID: e21e3c15-9fd1-47b2-a4bf-9c39c21d0fac
11:02:39 Task failed.

助手

查询任务结果后发现后端返回：

Model not initialized

原因是 API 曾用 --no-init 启动，服务能接任务但没有加载模型。于是移除了 Studio 启动脚本中的 --no-init，并固定使用更适合 8GB 显存的 acestep-5Hz-lm-0.6B。

用户 Insufficient KV cache

Generate
11:13:09 Submitting task...
11:13:09 Task ID: 959a7e64-8b6a-4a9b-99e6-7bfff0fce732
11:13:10 ERROR RuntimeError: Insufficient KV cache to schedule sequence.
Free blocks: 1/1, blocks needed: 2, prompt tokens: 33+30, block size: 256.

助手

判断该错误来自 8GB 显存下 nano-vllm 的 LM KV cache 不足。为了稳定生成，将 start_studio_api.bat 改为：

set "ACESTEP_LM_MODEL_PATH=acestep-5Hz-lm-0.6B"
set "ACESTEP_LM_BACKEND=pt"
set "ACESTEP_INIT_LLM=auto"

这样会绕开 nano-vllm KV cache 分配问题，速度可能慢一些，但更稳。
